# Color System

> The application uses semantic colors instead of hardcoded values.

---

# Primary Color

Purpose

- Primary buttons
- Active navigation
- Important actions
- Primary icons

---

# Secondary Color

Purpose

- Supporting text
- Secondary buttons
- Less important UI elements

---

# Accent Color

Purpose

- Badges
- Hover states
- Active indicators
- Decorative glows
- Small highlights

---

# White

Purpose

- Cards
- Containers
- Section backgrounds

---

# Black

Purpose

- Headings
- High emphasis text
- Primary content

---

# Border Color

Purpose

- Card borders
- Input borders
- Divider lines

Should always remain subtle.

---

# Success Color

Purpose

- Success notifications
- Form success state

---

# Error Color

Purpose

- Validation
- Failed actions

---

# Warning Color

Purpose

- Warning messages
- Informational notices

---

# Rules

Never use arbitrary colors.

Every UI element should use one of the semantic colors defined above.
