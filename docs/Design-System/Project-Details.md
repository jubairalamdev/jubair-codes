# Project Details Page Design Specification

> Purpose: Define the layout and design language of the Project Details page used to showcase an individual project.

---

# Overall Layout

The Project Details page expands upon a project selected from the Projects section.

The layout follows the same design language as the rest of the portfolio while providing additional information about a single project.

Characteristics

- Clean layout
- Spacious spacing
- Large floating content card
- Easy reading experience
- Strong visual hierarchy

---

# Page Structure

Content Order

1. Navigation Bar
2. Back Navigation
3. Project Details Card

---

# Navigation Bar

Use the same Navbar component documented in:

Navbar.md

---

# Back Navigation

Positioned directly below the navigation bar.

Purpose

Allow users to return to the Projects section.

Characteristics

- Small inline button
- Left aligned
- Minimal typography
- Small icon
- Subtle hover interaction

Example Placeholder

← Back to Projects

---

# Main Content Card

The entire project is displayed inside a large reusable card.

Characteristics

- Large border radius
- Thin subtle border
- White background
- Soft shadow
- Generous internal padding

---

# Card Header

Structure

- Decorative indicator
- Project Title
- Divider line

Example Placeholder

Portfolio Website

---

# Project Preview

Positioned below the title.

Characteristics

- Large hero image
- Rounded corners
- Fixed aspect ratio
- Full card width
- High quality preview

Purpose

Provide a visual overview of the project.

---

# Project Overview

Displayed directly below the preview image.

Purpose

Introduce the project.

Characteristics

- Comfortable reading width
- Secondary text
- Comfortable line height

Example Placeholder

A modern portfolio website built to showcase projects, technical skills, experience, and contact information with smooth animations and responsive design.

The remaining content is divided into clearly separated information blocks.

Each block contains:

- Section Heading
- Supporting Content

# Typography

Hierarchy

- Project Title
- Section Headings
- Body Text
- Technology Badges
- Button Text

---

# Spacing

Maintain generous spacing between:

- Navigation
- Back button
- Card sections
- Images
- Paragraphs
- Bullet lists
- Action buttons

The layout should encourage comfortable reading.

---

# Borders

Use

- Thin subtle borders
- Rounded cards
- Rounded technology badges
- Rounded buttons

---

# Shadows

Only the main content card uses subtle elevation.

Internal sections remain visually flat.

---

# Responsive Behaviour

Desktop

- Full-width reading layout
- Large preview image
- Comfortable content width

Tablet

- Reduced spacing
- Full-width preview image

Mobile

- Single-column layout
- Full-width content
- Stacked buttons
- Wrapped technology badges

---

# Design Principles

The Project Details page should communicate:

- Professionalism
- Technical depth
- Readability
- Organization
- Premium quality

Avoid

- Multiple columns inside the content
- Decorative graphics
- Heavy animations
- Visual clutter
- Long uninterrupted paragraphs
