# Projects Section Design Specification

> Purpose: Define the overall layout and structure of the Projects section.

---

# Overall Layout

The Projects section presents featured projects inside a large floating container while maintaining the same design language used throughout the portfolio.

Characteristics:

- Large rounded container
- Floating appearance
- Spacious layout
- Clean hierarchy
- Easy content scanning
- Grid-based layout

---

# Container

Properties:

- Large border radius
- Thin subtle border
- White background
- Soft shadow
- Generous internal padding

---

# Section Header

Structure:

- Decorative indicator
- Section title
- Divider line

Example:

Title:
Projects

---

# Category Filters

Positioned directly below the section header.

Purpose:

Allow visitors to filter projects based on technology or category.

Characteristics:

- Horizontal layout
- Pill-shaped buttons
- Equal spacing
- One active state
- Multiple inactive states

Example Categories

- All
- Full Stack
- Frontend
- Backend
- Next.js
- React
- Node.js

---

# Projects Grid

Desktop

- 3 Project Cards Per Row

Tablet

- 2 Project Cards Per Row

Mobile

- 1 Project Card Per Row

Cards should align consistently while maintaining equal spacing.

---

# Grid Spacing

Maintain generous spacing between cards.

Guidelines:

- Equal horizontal spacing
- Equal vertical spacing
- Consistent margins
- Balanced whitespace

---

# Responsive Behaviour

Desktop

Three-column layout.

Tablet

Two-column layout.

Mobile

Single-column layout.

Cards should stack naturally without breaking the visual rhythm.

---

# Design Principles

The Projects section should communicate:

- Professionalism
- Organization
- Creativity
- Modern design
- Easy navigation

Avoid:

- Uneven spacing
- Crowded layouts
- Decorative graphics
- Visual clutter
