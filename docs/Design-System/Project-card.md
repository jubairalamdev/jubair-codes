# Project Card Design Specification

> Purpose: Define the reusable project card component used throughout the portfolio.

---

# Overall Layout

The Project Card is a reusable component that highlights an individual project.

Structure:

1. Thumbnail
2. Badge
3. Project Title
4. Description
5. Technology Stack
6. Action Buttons

---

# Card Container

Properties:

- Large rounded corners
- Thin subtle border
- White background
- Soft shadow
- Comfortable internal padding

---

# Thumbnail

Positioned at the top.

Characteristics:

- Large preview image
- Fixed aspect ratio
- Rounded top corners
- Covers full width

---

# Project Badge

Displayed over the thumbnail.

Purpose:

Indicate project type.

Characteristics:

- Small pill badge
- Rounded corners
- Thin border
- Compact typography

Example Placeholders

- Featured Project
- Full Stack
- Frontend
- Backend
- Practice Project
- Open Source

---

# Project Title

Displayed below the thumbnail.

Characteristics:

- Medium-large typography
- Bold font weight
- Black text

Example Placeholder

Portfolio Website

---

# Project Description

Displayed below the title.

Characteristics:

- Comfortable line height
- Secondary text
- Limited number of lines
- Easy readability

Example Placeholder

A modern portfolio website showcasing projects, skills, experience, and interactive UI components.

---

# Technology Stack

Technologies are displayed as reusable badges.

Characteristics:

- Rounded pills
- Thin border
- Small typography
- Auto wrapping
- Equal spacing

Example Technologies

- HTML
- CSS
- JavaScript
- TypeScript
- React
- Next.js
- Node.js
- Express.js
- MongoDB
- Tailwind CSS

---

# Action Area

Located at the bottom of the card.

Structure:

Primary Button

- View Details

Secondary Icon Buttons

- Live Demo
- GitHub Repository

Characteristics:

- Rounded buttons
- Equal spacing
- Consistent sizing

---

# Typography

Hierarchy:

- Project Title
- Description
- Badge Text
- Technology Text
- Button Text

---

# Spacing

Maintain generous spacing between:

- Thumbnail
- Title
- Description
- Technology badges
- Action buttons

The action area should always remain aligned at the bottom of the card whenever possible.

---

# Borders

Use:

- Thin subtle borders
- Rounded corners
- Rounded badges
- Rounded buttons

---

# Shadows

Use only subtle elevation.

The card should appear to float slightly above the page.

---

# Hover Behaviour

The hover interaction should feel smooth and subtle.

Possible interactions:

- Slight elevation increase
- Slight upward movement
- Thumbnail zoom
- Accent color border
- Smooth transition

Avoid dramatic animations.

---

# Design Principles

The Project Card should communicate:

- Professionalism
- Technical expertise
- Modern design
- Simplicity
- High quality

Avoid:

- Heavy shadows
- Large decorative elements
- Excessive animations
- Visual clutter
